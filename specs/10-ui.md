# 10 — UI

Streamlit dashboard. Four panels + a subject input. Reads `state.json`
from the most recent run under `VOLLY_RUN_DIR`. Polls every ~1s while a
run is in progress.

## Layout

```
┌────────────────────────────────────────────────────────────────────┐
│ Volly — system prompt learning for ASCII art                       │
│ ┌──────────────────────────────────────────────────────────────┐   │
│ │ Subject: [ cat ▼ ]   Iterations: [ 8 ]   [ Run ]             │   │
│ └──────────────────────────────────────────────────────────────┘   │
├──────────────────────────────┬─────────────────────────────────────┤
│ Panel 1 — Evolving Prompt    │ Panel 2 — Current Best              │
│ (current text, with + / -    │ (rendered ASCII image, large)       │
│ diff vs. prior iteration)    │                                     │
├──────────────────────────────┼─────────────────────────────────────┤
│ Panel 3 — Win Rate           │ Panel 4 — Judge Reasoning           │
│ (two lines: evolving vs.     │ (latest critique + suggestions      │
│ control, x = iteration)      │ in plain English)                   │
└──────────────────────────────┴─────────────────────────────────────┘
```

## Behavior

- **Subject input:** dropdown of the curated list defined in
  `specs/00-overview.md`. Free-text input sanitized to a curated subject
  (closest match) before the loop receives it.
- **Run button:** spawns the loop as a background task via
  `concurrent.futures` (Streamlit's session state holds the future). The
  page polls `state.json` and rerenders.
- **Panel 1:** shows the latest evolving prompt. Below it: a unified diff
  vs. the previous iteration's prompt, with red `-` and green `+` lines.
- **Panel 2:** the best-of-N image from the latest evolving iteration.
  Update on every state.json change.
- **Panel 3:** a Streamlit chart (`st.altair_chart` — `st.line_chart`
  cannot bound the y-axis in current Streamlit) with two series —
  evolving and control win rate per iteration. Y-axis hard-bounded to
  [0, 1] so a flat control arm visually reads flat against the climbing
  evolving arm. Color: evolving green, control gray, so the demo
  narrative ("the system learning vs. the frozen baseline") is legible
  to the audience without a legend lookup.
- **Panel 4:** the latest `judge.critique` paragraph + bulleted
  `prompt_suggestions`. Plain text, large enough to read on a projector.

## Run state machine

```
idle → running → finished (or errored)
```

While `running`, the Run button is disabled and replaced with a Stop
button. Stop simply cancels the future and writes a partial `state.json`.

## Read-only mode

If `state.json` exists at startup, the UI opens in read-only mode showing
the most recent finished run. No background loop spawned until a new Run
is clicked.

## Why Streamlit, not Next.js

Decided in `specs/01-stack.md`. Four panels of pre-rendered images and
text in Python — Streamlit is the right tool. Next.js would be a hackathon
self-own.

## Test surface

UI is not unit-tested in v1. Manual smoke test via:
```
streamlit run volly/ui/app.py
```
plus a finished `runs/<...>/state.json` to load.
